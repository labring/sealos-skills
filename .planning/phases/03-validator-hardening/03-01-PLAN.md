<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repository docs and metadata to maintainer gate | README and JSON metadata are untyped repository inputs consumed by `scripts/validate-codex-plugin.py`. |
| Maintainer gate to publishing workflow | Maintainers rely on one local command before publishing Codex-facing distribution changes. |
| Metadata to user install path | README and registry fields tell users which repository/plugin to install and how to invoke it. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | README and platform install commands | mitigate | Assert exact repository slug `labring/sealos-skills`, selector `sealos@sealos`, and canonical install/fallback command strings in `scripts/validate-codex-plugin.py`. |
| T-03-02 | Tampering | JSON metadata files | mitigate | Parse `.codex-plugin/plugin.json`, root `plugin.json`, `.agents/plugins/marketplace.json`, root `marketplace.json`, and `distribution/platforms.json` through `load_json()` and verify with `python3 -m json.tool`. |
| T-03-03 | Repudiation | Validation evidence | mitigate | Task 3 records positive validator output and targeted negative failure probes in the execution summary. |
| T-03-04 | Information Disclosure | Platform support claims | mitigate | Keep checks Codex-focused and assert exact Codex invocation/evidence text so registry copy matches verified support. |
| T-03-05 | Denial of Service | Validator parser behavior | mitigate | Preserve existing single-command Python standard library path with concise failures and no external package dependency. |
| T-03-06 | Elevation of Privilege | Runtime skill behavior | accept | Phase only reads repository files and does not change auth, deploy, database, S3, or canvas runtime behavior; verify with `git diff -- skills --exit-code`. |
| T-03-SC | Tampering | package installs | mitigate | No npm, pip, or cargo installs are required; keep the phase on Python standard library tooling. |
</threat_model>
